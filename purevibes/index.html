Why It Hits

A local-library player with taste, speed, and native restraint.

“Most music players treat local libraries as a downgrade from streaming. PureVibes treats them as the main event.”

That attitude drives everything here: OLED-black presentation, artwork-led browsing, responsive queue controls, and a player that feels native because it is native.

01

Album art does the talking

Rounded covers, cinematic spacing, glass surfaces, and elegant serif typography make the UI feel closer to a collector’s shelf than a spreadsheet.

02

Waveforms without the lag

RMS waveform maps are computed once, cached to CoreData, and reused instantly so scrub feedback feels immediate even in large libraries.

03

Metadata with real depth

Apple Digital Master detection, advisory badges, bitrate quality tiers, and artwork fallbacks all surface the kind of detail library obsessives actually notice.

Playback

Queue management that behaves like a desktop app.

Drag-to-reorder, play-next, add-to-end, loop modes, shuffle recovery, and media key support keep playback control close at hand without bloating the main view.

Metadata

Apple Digital Master detection built from raw atoms.

Instead of hand-wavy labels, PureVibes checks `flvr`, `atID`, `cnID`, and ownership markers so surfaced metadata is tied to real file information.

Performance

Large libraries stay smooth by design.

Adaptive performance modes, thermal-aware caching, and lazy metadata resolution keep the UI responsive as collections scale up.

Architecture

Under the glass, the engineering is as deliberate as the UI.

System Notes

  • Artwork cache: full-res cache, thumbnail cache, and dominant-color memory layer
  • Waveform engine: chunked PCM sampling with persisted RMS energy maps
  • Contrast logic: ITU-R BT.709 luminance for readable overlays on artwork
  • Library access: security-scoped bookmarks for multi-folder persistence
  • Track resolution: bounded LRU metadata loading for better scaling

Built With

SwiftUI, AVFoundation, CoreData, and MediaPlayer.

No browser shell. No web view wrapper. Just a focused native stack tuned for music playback, artwork handling, and Now Playing integration on macOS.
